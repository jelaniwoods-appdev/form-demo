Keyword Description Example
true, false Conditional program execution: program starts with if branch, tries the elsif branches, and finishes with else branch (until one branch evaluates to else) 

ruby_is_the_best = true
squares_have_three_sides = false
if, elsif, else An if statement evaluates an expression, which returns either true or false. When true, Ruby executes the "code block" that follows. When the expression is false, Ruby returns nil. In this example, the string "Your condition was true!" will print because the condition number == 10 is true. 

number = 10
if number == 10
  puts "Your condition was true!"
end
nil nil is a special Ruby object used to represent an "empty" or "default" value. Many methods can return nil as a result. This happens when you ask for a value but that value is not available:
  • Array index out of bounds
  • Hash key or value doesn't exist
  • Undefined instance variables
  • Record not found in our database 

letters = ["a", "b", "c"]
letters[5]
# => nil
|| The || (pronounced "or") operator is a logical operator which returns true if either of the expressions on left-hand side or right-hand side evaluate to true. 

grade1 = 50
grade2 = 30
grade3 = 80

if grade1 > grade2 || grade1 > grade3
  p "Grade 1 is not the lowest score!"
end
&& The && operator (pronounced "and") is a logical operator in which evaluates to true only if both expressions on either side of && evaluates to true. 

if score1 > score2 && score1 > score3
  print "Score 1 is the greatest in value."
else
  print "Score 1 is not the greatest in value."
end
== A logical operator in which compares the expressions on both sides and returns true is both values are the same. Otherwise, it returns false. 

age = 30
if age == 15 * 2
  p "age is the same as 15 * 2"
end
def Defines a method. A method is a block of code that can easily be run by invoking the method name. A def requires a method name and an end to denote where the method ends. All methods must be called on some object. 

class Greeting
  def say_hello
    p "Hello!"
  end
end

g = Greeting.new
g.say_hello
# => "Hello!"
return A keyword that allows the developer to explicitly stop the execution flow of a method and return a specific value. Note: values that are returned are not printed by 

class Addition
  def add_two(number)
    return number + 2
  end
end

add = Addition.new
result = add.add_two(3)
# => 5
class A class is the blueprint from which individual objects are created. Classes allow us to write methods and attributes which can be reused by many objects. 

class Fruit
  attr_attribute(:name)
  attr_attribute(:color)
end

strawberry = Fruit.new
strawberry.name = "Strawberry"
strawberry.color = "red"
